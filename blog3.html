The MVC Architecture

Why Everyone Talks About MVC in AngularJS

When you first start building with AngularJS, everything feels simple… until your code slowly turns into a confusing mess. Variables here, functions there, HTML mixed with logic — and suddenly, debugging becomes a nightmare.

This is exactly the problem AngularJS solves using the MVC pattern. It stands for Model, View, Controller — and once you understand this, AngularJS starts making a lot more sense.

Breaking Down MVC (In Simple Words)

Instead of putting everything in one file, Angular divides your app into three distinct parts:

ModelHolds your data
ViewDisplays your data
ControllerManages the logic

The Model: Where Your Data Lives
In AngularJS, the Model is mainly handled using the $scope object. Think of it as a storage box. When you say $scope.name = "Sonali";, Angular makes that data available everywhere.

The View: What the User Sees
The View is your HTML, but with superpowers. Instead of hardcoding text, you use {{ name }}. Angular then says: "Take the value from the Model and show it here."

The Controller: The Brain Behind It
The Controller is the connector. It defines the data, stores it in the Model ($scope), and the View automatically displays it.

Try It Yourself: The MVC Flow

Observe how the Controller sets the data and the View simply "asks" for it. This separation keeps your project from turning into chaos.

Code Playground

// 1. THE CONTROLLER (Logic)
app.controller("myCtrl", function($scope) {
    // 2. THE MODEL (Data)
    $scope.name = "Sonali"; 
});

<!-- 3. THE VIEW (Display) -->
<div ng-controller="myCtrl">
    <h1>Welcome, {{ name }}</h1>
</div>

A Simple Way to Remember

👉 Controller tells Model what to store
👉 View shows what Model has
