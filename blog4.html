Powering HTML with Directives

What If HTML Could Do More?

Normally, HTML is pretty simple. You write tags, display content… and that’s it. It doesn’t “think,” it doesn’t react — it just shows what you tell it. But AngularJS changes that by giving HTML superpowers using Directives.

Directives are special markers placed inside HTML that tell AngularJS to attach a specific behavior to that element. They look like normal attributes, but they carry a world of logic behind them.

The Core Directives

ng-app: The Engine

Every app needs a starting point. This tells Angular: "Everything inside this tag belongs to my app." Without it, the engine won't turn on.

ng-model: The Real Magic

This creates a live connection between your input box and your data. It enables Two-Way Data Binding: Input updates data → Data updates UI instantly.

ng-bind: Professional Display

A cleaner way to show data. It works like {{ }} but prevents that quick "flash" of raw code before the page fully loads.

Try It Yourself: The Live Connection

Copy this into your index.html. Watch how ng-init sets the starting point and ng-model keeps the conversation going.

// DIRECTIVE PLAYGROUND 
<div ng-app="" ng-init="name='Sonali'">
    <p>Enter Name: <input type="text" ng-model="name"></p>
    
    <!-- Using two different ways to display -->
    <h3>Hello {{ name }}</h3>
    <p ng-bind="name"></p>
</div>

Why It Matters

Before AngularJS, you had to write heavy JavaScript to update the screen every time a user typed a letter. With directives, your HTML is no longer "dumb"—it reacts, updates, and responds with zero manual DOM manipulation.
